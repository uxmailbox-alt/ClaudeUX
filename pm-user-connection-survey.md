# Survey: How PMs Connect With Users & the Field

**Purpose (for context, not part of the form):** Understand how PMs actually stay connected to
users today — channels, cadence, diversity, and open feedback capture — in order to evaluate
whether the monthly Product Advisors program (a small, cross-product pool of users) is still the
right vehicle, or whether it should be reshaped/replaced. The program exists because PMs used to
be dependent on CS to get any user time; this survey checks whether that constraint still holds.

Copy the sections below into Google Forms. Suggested question types are noted in `[brackets]`.
Use Forms' "Go to section based on answer" feature for the two branch points called out below.

## Notes on analysis

- Q3 + Q4 + Q12 together tell you whether the original "we're dependent on CS" problem still
  exists — if most PMs report frequent direct access and low CS dependency, that's the strongest
  signal the program's founding rationale has weakened.
- Q6 + Q7 directly test your hypothesis that the pool is too diverse for a fixed cadence — a
  cluster of "fixed pool, low relevance" answers validates moving toward ad-hoc/topic-based
  pods.
- Q11 (topic prep difficulty) is a leading indicator of the same fatigue you're already sensing.
